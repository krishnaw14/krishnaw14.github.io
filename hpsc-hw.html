High Performance Scientific Computing

ME766 (Course on High Performance Scientific Computing) had 3 major assignments which aimed at getting familiar with parallel programming (openMP, MPI and opencl/cuda) and code profiling and debuggin tools (gdb, gprof, valgrind).

I parallelised the serial code for trapezoid and montecarlo integration using the aforementioned parallel programming tools. I was able to speed the code upto 2 times with openmp, 6 times with with MPi and around 1000 times with opencl (with reference to serial code). In case of opencl code, time evaluaiton was done after the GPU device was found so the actual time might be marginally more than what was observed.The GPU code had been optimised with respect to the number of work groups, that have been formed based on the work group size and no of sampling points provided. The reports attached demonstrate how good GPUs are with respect to scaling a problem.

The 3rd assignment required detailed profiling of the code analysing the call graph and hit/miss rates.


The Final report, codes, profing output and visualization modules can be found in the github repository: HPSC-Homeworks.
